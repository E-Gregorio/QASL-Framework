Que son las Pruebas Estaticas?

Definicion ISTQB: Pruebas que se realizan SIN ejecutar el codigo. Se analizan documentos, requisitos e historias de usuario de forma estatica para encontrar defectos temprano.

Que Analizamos?

  • Historias de Usuario (HUs)
  • Criterios de Aceptacion
  • Reglas de Negocio (BRs)
  • Escenarios de Prueba
  • Documentos de Requisitos

Que Buscamos?

  • Ambiguedades en requisitos
  • Escenarios no contemplados
  • Reglas de negocio sin cobertura
  • Inconsistencias logicas
  • Gaps de seguridad

HU_SGINC_05 - Carga Individual

Historia de Usuario Analizada

Historia de Usuario

"Como usuario con perfil CARGA, quiero acceder a un formulario de carga individual desde la grilla de inconsistencias, para registrar datos de contribuyentes de forma manual."

Reglas de Negocio Identificadas (4 BRs)

BR1: Acceso al formulario desde la grilla de inconsistencias BR2: Formulario contiene campos segun diseno Figma (CUIT, Razon Social, etc.) BR3: Error "Faltan campos obligatorios" si datos incompletos BR4: Al guardar, estado = "Ingresada" y origen = "Carga Manual"
