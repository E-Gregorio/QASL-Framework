Shift-Left Testing

Que es Shift-Left Testing?

Definicion: Practica de mover las actividades de prueba hacia etapas mas tempranas del ciclo de desarrollo. "Hacia la izquierda" en la linea de tiempo.

Enfoque Tradicional

  • Pruebas al final del desarrollo
  • Defectos encontrados tarde
  • Costo de correccion alto
  • Retrasos en entregas

Enfoque Shift-Left

  • Pruebas desde requisitos
  • Defectos encontrados temprano
  • Costo de correccion minimo
  • Entregas predecibles

Flujo de Trabajo

Flujo Completo de Trabajo QA

FASE 1: Pruebas Estaticas
Analisis de HUs → Identificacion de BRs → Generacion de TCs → CSVs de Trazabilidad
FASE 2: Deploy del SUT
Sistema Bajo Prueba desplegado → Ambiente de Test preparado
FASE 3: Pruebas Exploratorias
Ejecucion basada en TCs → Refinamiento de HUs → Deteccion de defectos
FASE 4: Backlog y Planning Poker
HUs al backlog → Estimacion VCR → Votacion del equipo
FASE 5: Automatizacion (VCR >= 9)
E2E con Playwright → API con Postman/Newman → Performance con K6 → Seguridad con OWASP ZAP

Priorizacion

Priorizacion: VCR y Planning Poker

Formula VCR

VCR = Valor + Costo + Riesgo

Escala: 1 (bajo) a 3 (alto) por cada dimension | Rango: 3 a 9

Valor

Impacto para el negocio y el usuario final

Costo

Esfuerzo de desarrollo y pruebas

Riesgo

Probabilidad e impacto de fallo

Si la votacion es VCR >= 9 → Candidata a automatizacion y se convierte en deuda tecnica

Pruebas de Performance

K6
Pruebas de carga y estres

Tipos de Prueba

  • Load Testing: Carga normal
  • Stress Testing: Limites del sistema
  • Spike Testing: Picos subitos
  • Soak Testing: Carga sostenida

Metricas Clave

  • P95: Percentil 95 tiempo respuesta
  • Error Rate: Tasa de errores
  • Throughput: Requests por segundo
  • VUs: Usuarios virtuales

Umbrales Sugeridos

P95 < 2 segundos

Error Rate < 1%
